Reverse Engineering Report: TASCAM US-144 MKII USB Audio Protocol

Document Version: 13.1 (Updated with Capture & Control Logic)
Date: July 2, 2025
Subject: This document provides a complete and re-verified specification for the TASCAM US-144 MKII's audio streaming and control mechanisms across all supported sample rates, based on extensive reverse-engineering of the official macOS and Windows drivers, including dynamic USB sniffing data and static USB device descriptor reports. It is intended as a precise technical reference for implementing a functional Linux driver.
1. Core Architecture

The device operates as a native 4-in, 4-out audio interface at the USB transport layer. Playback is achieved via a host-paced asynchronous isochronous feedback loop, while capture is handled by a parallel bulk transfer stream.

    Audio Data (OUT): Endpoint 0x02. The host sends a continuous 4-channel, 24-bit interleaved audio stream.

    Clock Feedback (IN): Endpoint 0x81. The device sends its master clock rate to the host, allowing the driver to correct for playback drift.

    Capture Data (IN): Endpoint 0x86. The device sends a continuous 4-channel, 24-bit interleaved audio stream from its inputs to the host via Bulk transfers.

    MIDI I/O: Endpoints 0x04 (OUT) and 0x83 (IN) provide standard MIDI functionality.

3. Playback (OUT) Stream (Endpoint 0x02)

    URB Structure: number_of_packets: 40, transfer_buffer_length: 2880 bytes (nominal).

    Packet Structure: Each of the 40 packets has a dynamic size determined by the "Packet Fixing" engine. packet_size_bytes = frames_for_this_packet * 12 (4 channels * 3 bytes/sample).

    Audio Data Format: The driver must construct a 4-channel, 24-bit interleaved stream. The mapping of the 2-channel user audio to the 4 device channels is controlled by hardware routing settings. The default behavior is to map L/R to channels 1/2 and send silence on channels 3/4.

4. Capture (IN) Stream (Endpoint 0x86)

    URB Structure: Standard Bulk URBs. The driver should submit a pool of 3-4 URBs to ensure a continuous queue, mirroring the PGDevice::bulkAudioGo logic.

    Data Format: Raw, headerless stream of 12-byte frames.

    Frame: [C1_B1..B3, C2_B1..B3, C3_B1..B3, C4_B1..B3] (24-bit, little-endian).

    Decoding: The pcmFrom24LSBChannelSwapMI_4 logic must be used to de-interleave, re-order, and pad the samples to a 4-channel, 32-bit format for ALSA.

5. Feedback (IN) Stream (Endpoint 0x81) & "Packet Fixing" Synchronization Engine

This is the core clock recovery mechanism, implemented as a digital Phase-Locked Loop (PLL).

    Feedback Packet Format: 3 bytes. The first byte contains the feedback value (e.g., 0x30 for 48 frames/ms).

    URB Structure (Latency-Dependent): The number of packets per URB varies by latency setting.
    | Latency Setting | Feedback URB number_of_packets |
    | :--- | :--- |
    | Lowest/Low | 1 |
    | Normal | 2 |
    | High/Highest | 5 |

        Note: The bInterval for EP 0x81 is 0x04 (1 ms). The varying feedback intervals are achieved by changing the number_of_packets in the URB.

    State Variables (for tascam_card struct):

        feedback_accumulator_pattern[128]: Circular buffer holding frames-per-microframe values.

        feedback_pattern_out_idx, feedback_pattern_in_idx: Read/write indices for the accumulator.

        feedback_synced: Boolean flag for PLL lock status.

        playback_frames_consumed: u64 counter for the hardware pointer.

    Initialization: The accumulator is filled with the nominal frames per microframe (e.g., 6 for 48kHz). Indices are zeroed, feedback_synced is false.

    Pattern Tables: Pre-calculated lookup tables mapping feedback values to 8-entry microframe patterns.
    | Sample Rate | Feedback Value | Pattern Table (8 entries, sum in parentheses) |
    | :--- | :--- | :--- |
    | 48 kHz | 46 (0x2E) | {5, 6, 6, 6, 5, 6, 6, 6} (46) |
    | | 47 (0x2F) | {5, 6, 6, 6, 6, 6, 6, 6} (47) |
    | | 48 (0x30) | {6, 6, 6, 6, 6, 6, 6, 6} (48) |
    | | 49 (0x31) | {7, 6, 6, 6, 6, 6, 6, 6} (49) |
    | | 50 (0x32) | {7, 6, 6, 6, 7, 6, 6, 6} (50) |

        Note: Tables for other rates require further analysis of the PGFramePatternObserver::init function.

    Feedback Processing (feedback_urb_complete):

        On URB completion, extract feedback_value from each packet.

        If valid, look up the corresponding 8-entry pattern and write it to the feedback_accumulator_pattern at the in_idx.

        Advance in_idx.

        Update playback_frames_consumed by the sum of frames just added to the accumulator.

        Resubmit the feedback URB.

        Call snd_pcm_period_elapsed() when playback_frames_consumed crosses a period boundary.

    Playback URB Preparation (playback_urb_complete):

        For each of the 40 packets in the playback URB, read the next value from feedback_accumulator_pattern at the out_idx to get frames_for_this_packet.

        Advance out_idx.

        Calculate packet_size_bytes = frames_for_this_packet * 12.

        Set the URB's iso_frame_desc length and offset.

        Copy the calculated number of frames from the ALSA buffer, performing the 2-to-4 channel mapping.

        Resubmit the playback URB.

6. ALSA Pointer Callback (tascam_pcm_pointer)

    Playback: Returns tascam->playback_frames_consumed % runtime->buffer_size.

    Capture: Returns the value of a similar capture_frames_consumed counter, which is incremented in the capture_urb_complete callback.
